Explicando o GitHub

O GitHub é uma plataforma de hospedagem de código-fonte e colaboração baseada em controle de versão usando o Git. O Git é um sistema de controle de versão distribuído, o que significa que permite rastrear mudanças no código ao longo do tempo e facilita o trabalho colaborativo em projetos de software.

Manipulando arquivos

  • Configuração inicial
      • 1º Instalar o Git no seu computador, se ainda não estiver instalado.
      2º Configurar sua identidade localmente no Git usando git config.
  • Criar um repositório local
      • Inicializar um novo repositório Git em seu computador usando git init na pasta do seu projeto ou clonar um repositório existente do GitHub usando git clone URL_DO_REPOSITORIO.
  • Adicionar e modificar arquivos
      • Quando você cria um novo arquivo, o Git não o rastreia automaticamente; você precisa adicioná-lo usando git add NOME_DO_ARQUIVO ou git add . para adicionar todos os arquivos modificados.
    • Commit das alterações
        • 1º Após adicionar os arquivos, você os prepara para serem incluídos no histórico do Git por meio de um "commit".
        2º Use git commit -m "MENSAGEM_DO_COMMIT" para confirmar as alterações com uma mensagem descritiva.
    • Enviar para o repositório remoto no GitHub
        • 1º Associe seu repositório local a um repositório remoto no GitHub usando git remote add origin URL_DO_REPOSITORIO.
        2º Envie suas alterações para o repositório remoto usando git push origin BRANCH(normalmente main ou master) para enviar os commits para o GitHub.
    • Atualizações futuras
        • - Se outros colaboradores fizerem alterações no repositório remoto, você pode trazer essas alterações para o seu repositório local usando git pull origin BRANCH.
